## Key Decisions

1. **`.slnx` instead of `.sln`**: .NET 10 SDK `dotnet new sln` generates the new XML solution format (`.slnx`). The plan specified `MongoDbPatterns.sln` but `.slnx` is the native format. All `dotnet` CLI commands work identically.
2. **`nuget.config` added**: Machine-level NuGet configuration had private Azure DevOps feeds causing `NU1301` restore errors. A repo-level `nuget.config` with `<clear />` + `nuget.org` only was added to isolate the project.
3. **MongoDB.Driver 3.8.0**: Latest version at time of install. Note: this is the v3 driver which has breaking API changes from v2 (e.g., `MongoClient` API changes, `BsonDocument` handling). Future tasks implementing repositories should use v3 API patterns.
4. **BenchmarkDotNet 0.15.8**: Latest stable version.
